Unix gave us `stdin`, `stdout`, and `stderr`. stdai adds the fourth stream: **durable work that survives beyond the immediate response.**

Every research finding, plan, decision, and handoff your agents produce gets stored as a cryptographically signed, searchable artifact — with full lineage back to what it was based on.

```bash
python3 research.py | stdai write --kind research --tag security | python3 fact_checker.py
```

## Why stdai?

Agentic systems produce work that doesn't fit neatly into Unix's three streams: research findings, plans, handoffs, fact checks, decisions, investigation notes. This work needs to survive across sessions and agents, be searchable, and have clear provenance.

## Get started in 3 steps

<Steps>
  <Step title="Install stdai">
    ```bash
    curl -sSf https://raw.githubusercontent.com/ooboai/stdai/main/install.sh | bash
    ```
  </Step>
  <Step title="Create a signing identity">
    ```bash
    stdai identity new --label "my-agent"
    export STDAI_IDENTITY=stdai:a3b7c9d8e1f2...
    ```
  </Step>
  <Step title="Write your first artifact">
    ```bash
    stdai write --kind note --content "guest sessions break when middleware assumes user role"
    ```
  </Step>
</Steps>
