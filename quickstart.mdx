This guide walks you through installing stdai, creating a signing identity, and writing your first artifact.

## Prerequisites

- stdai installed — see [Installation](/installation) if you haven't done this yet

## Step-by-step

<Steps>
  <Step title="Create a signing identity">
    All writes must be signed. Create your first identity:

    ```bash
    stdai identity new --label "my-agent"
    ```

    You'll see output like:

    ```
    Created identity:
      address:  stdai:a3b7c9d8e1f2...
      label:    my-agent
    ```

    The address is your identity's unique identifier — an `stdai:` prefix followed by 40 hex characters derived from your Ed25519 public key. The private key is stored at `~/.stdai/identities/<address>/secret.key`.
  </Step>

  <Step title="Set your identity for the session">
    Export the address so stdai signs all writes automatically:

    ```bash
    export STDAI_IDENTITY=stdai:a3b7c9d8e1f2...
    ```

    <Tip>
      Add this to your shell profile or agent startup script so it's always available. You can also pass `--identity <address>` per-command if you prefer.
    </Tip>
  </Step>

  <Step title="Write your first artifact">
    Store a note with a kind label:

    ```bash
    stdai write --kind note --content "guest sessions break when middleware assumes user role"
    ```

    stdai prints the artifact ID to stdout:

    ```
    01JVQX3M4K5N8P2R7T9W0YABCD
    ```

    <Note>
      The artifact ID is a ULID — a timestamp-sortable, globally unique identifier. You can use the first few characters as a prefix to reference it later.
    </Note>
  </Step>

  <Step title="Write an artifact via pipe">
    stdai works as a transparent pipe — it captures stdin as an artifact and forwards the bytes to stdout unchanged:

    ```bash
    echo "research output" | stdai write --kind research | next-step.sh
    ```

    The artifact ID is emitted to stderr so it doesn't interfere with your pipeline. To suppress forwarding (capture only), use `--no-forward`:

    ```bash
    echo "capture only" | stdai write --kind note --no-forward
    ```
  </Step>

  <Step title="Search for your artifact">
    Find artifacts by full-text query (scoped to the current project by default):

    ```bash
    stdai find "guest session"
    ```

    Filter by kind or tag:

    ```bash
    stdai find --kind note
    stdai find --tag security
    ```

    Search across all projects:

    ```bash
    stdai find "guest session" --all
    ```
  </Step>

  <Step title="Inspect an artifact">
    Show full detail for any artifact (prefix matching works):

    ```bash
    stdai show 01JVQX3M
    ```

    Get just the raw content:

    ```bash
    stdai show 01JVQX3M --content-only
    ```

    Verify the signature:

    ```bash
    stdai verify 01JVQX3M
    ```
  </Step>
</Steps>
